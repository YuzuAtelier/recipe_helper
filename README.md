# レシピヘルパー(recipe_helper)

冷蔵庫にある食材を **1〜5つ** 選ぶと、一致度の高いレシピをおすすめしてくれる個人用PWAアプリです。
完全無料・完全オフライン動作(外部API・通信なし)。

## 特徴

- **食材から探す**: 5カテゴリ(野菜/肉・魚/乳製品/加工食品/主食・粉類)から食材をタップして選択
- **精度の高いマッチング**: 選択食材とレシピの主要食材タグの一致数でスコアリング
  - 1個選択: 1つ一致するレシピを表示
  - 2個以上選択: 選択食材のうち **2つ以上を含む** レシピを優先表示(少ない場合は1つ一致を補足表示)
- **絞り込み**: 調理時間(10/15/30分以内)・調理法(焼く/炒める/煮る…)で副菜も探しやすい
- **お気に入り**: 端末内(localStorage)に保存
- **PWA**: ホーム画面に追加してアプリのように使え、オフラインでも動作

## データ規模(品質チェック済み)

- レシピ **298件** / 食材マスタ **90品**
- 全食材が主要食材として3件以上のレシピに登場
- 日常的な定番ペアは「両方を含む」レシピが3件以上

## フォルダ構成

```
recipe_helper/
├── index.html          ← アプリ本体(単体HTML)
├── manifest.json       ← PWAマニフェスト
├── sw.js               ← Service Worker(オフライン対応)
├── .nojekyll           ← GitHub Pages用
├── icons/              ← アプリアイコン(192/512px)
├── data/
│   ├── ingredients.js  ← 食材マスタ(90品)
│   └── recipes.js      ← レシピデータ(298件)
├── tools/
│   └── validate.js     ← データ整合性チェック(node tools/validate.js)
└── docs/
    ├── REQUIREMENTS.md ← 要件定義
    └── DECISIONS.md    ← 決定事項・検討経緯
```

## ローカルでの動作確認

`data/*.js` は `<script src>` で読み込むため、ファイルを直接開いても動きますが、
Service Workerを含めて確認する場合はローカルサーバー経由が確実です。

```bash
npx http-server -p 8123 -c-1 .
# → http://localhost:8123 を開く
```

## レシピデータの検証

```bash
node tools/validate.js
```

タグの表記ゆれ(食材マスタとの不一致)、id重複、調理法・ジャンルの整合性、
食材・ペアのカバレッジを自動チェックします。
