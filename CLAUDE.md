# CLAUDE.md — レシピヘルパー(recipe_helper)

このファイルはClaude Codeがプロジェクト開始時に読む指示書です。

## プロジェクト概要

アプリ名: **レシピヘルパー**(英字スラッグ: `recipe_helper`)

食材を3〜5つ選択すると、一致度の高いレシピを3つ以上おすすめしてくれる個人用PWAアプリ。

## 開発者(ユーザー)について

- GitHubアカウント: YuzuAtelier
- 単体HTML/PWA + GitHub Pages での公開経験あり(前作: ichiban-uranai, sukusuku-note, tabi-saifu)
- ステップごとに確認しながら進める進め方を好む

## 進め方のルール(重要)

1. **必ずステップを最初に全て提示し、1ステップずつユーザーの確認を取ってから進めること**
2. 勝手に次のステップへ進まない
3. レシピデータの作成は件数が多いため、ジャンルごとにバッチで作成し、都度サンプルを提示して品質を確認してもらうこと(一度に全件作らない)
4. コミット・プッシュ・デプロイなどの操作は実行前に必ず確認を取る

## 技術方針

- **形式**: 単体HTMLファイルのPWA(Service Worker + manifest は別ファイル可)
- **ホスティング**: GitHub Pages(YuzuAtelierアカウント)
- **データ保存**: レシピデータは静的JSON(アプリに同梱、外部API不使用)。お気に入り等のユーザーデータはlocalStorage
- **完全無料・完全オフライン動作**が絶対条件
- **外部ライブラリ**: 最小限に。CDN依存はオフライン動作を壊さない範囲で

## 最優先事項: マッチング精度

このアプリの価値はマッチングの精度そのものにあります。件数を増やすことより、以下を優先してください。

- レシピごとの `main_ingredients`(主要食材タグ)を、ユーザーが選びそうな食材の言葉遣いに正確に合わせる(表記ゆれに注意: 「鶏もも肉」「鶏もも」など揺れが出ないよう、選択用の食材マスタと完全一致させる)
- 同じ食材でも切り方・部位表記で分裂しないよう、食材マスタ(選択肢として提示する食材の正式名称リスト)を先に確定してから、そのマスタの語彙だけでレシピにタグ付けする
- ジャンル・調理法(焼く/煮る/蒸す/生食/汁物など)に幅を持たせ、同じ食材セットでも複数の系統のレシピが出せるようにする
- 件数目標(数百件)よりも、食材マスタとのタグ整合性を優先する。整合性が取れないレシピは無理に件数に含めない

## フォルダ構成

- `docs/REQUIREMENTS.md` — 詳細要件定義
- `docs/DECISIONS.md` — これまでの検討経緯と決定事項
- `PROMPT.md` — 開発開始時にユーザーがClaude Codeに貼る最初のプロンプト

## 最初にやること

1. `docs/REQUIREMENTS.md` と `docs/DECISIONS.md` を読む
2. 食材マスタ(5カテゴリ×選択可能な食材の正式名称リスト)の案を提示し、ユーザーの確認を取る
3. 食材マスタが確定したら、レシピデータのJSONスキーマを提示し確認を取る
4. ジャンルごとに少量ずつレシピを作成し、都度サンプルを見せて品質を確認してもらう
5. 上記が固まってから、画面(食材選択→一覧→詳細→お気に入り)の実装に進む
