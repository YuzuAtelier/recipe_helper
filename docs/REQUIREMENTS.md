# 要件定義 — レシピヘルパー(recipe_helper)

## 1. 目的

冷蔵庫にある食材を3〜5つ選ぶだけで、作れるレシピを精度高くおすすめしてくれる個人用アプリ。

## 2. 利用者と利用形態

- 個人利用(開発者本人)
- PWA。ホーム画面に追加してアプリのように使う
- 完全オフライン動作(レシピデータは同梱、外部通信なし)

## 3. 食材カテゴリ(確定)

① 野菜(きのこ含む)
② 肉・魚
③ 乳製品(卵含む)
④ 加工食品(豆腐・練り物・ハム・缶詰など)
⑤ 主食・粉類(米・パスタ・パン・小麦粉など)

## 4. 機能要件

### 4.1 食材選択
- カテゴリタブ(5つ)を切り替えながら、食材をタップして選択
- 3〜5個選択したら「レシピを探す」ボタンが有効になる
- 選択中の食材はいつでも確認・解除できる

### 4.2 レシピのおすすめ
- 選択した食材と各レシピの `main_ingredients` の一致数でスコアリング
- 一致数が多い順に上位を表示。最低3件、可能な限りそれ以上表示
- 一致数が同じ場合は調理時間が短い順
- 一覧では: レシピ名、ジャンル、調理時間、一致した食材のハイライトを表示

### 4.3 レシピ詳細
- 材料(主要食材+その他材料)、調理時間、人数、手順を表示
- お気に入り登録ボタン

### 4.4 お気に入り
- お気に入りに登録したレシピを一覧できる画面
- localStorageに保存

## 5. レシピデータの要件(最重要)

- **精度優先**: マッチング精度を最優先とし、件数のために品質を落とさない
- 食材マスタ(選択肢として提示する食材の正式名称リスト)を先に確定し、その語彙のみで `main_ingredients` にタグ付けする(表記ゆれの防止)
- ジャンル・調理法に幅を持たせる(和食/洋食/中華/作り置き・時短/スープ・鍋 等、偏りなく)
- レシピ数は数百件を目指すが、目安であり、整合性を犠牲にしてまで件数を優先しない

### データ構造

```json
{
  "id": "recipe_001",
  "name": "鶏もも肉の照り焼き",
  "genre": "和食",
  "main_ingredients": ["鶏もも肉", "醤油", "みりん"],
  "other_ingredients": ["酒", "砂糖", "サラダ油"],
  "time_minutes": 20,
  "servings": 2,
  "steps": ["...", "..."]
}
```

## 6. 非機能要件

- 単体HTML中心の軽量構成(Service Worker / manifest / アイコンは別ファイル可)
- 完全無料・完全オフライン(外部API・通信不使用)
- 日本語UI

## 7. 画面構成

1. 食材選択(カテゴリタブ→3〜5個選択)
2. おすすめレシピ一覧(一致度順)
3. レシピ詳細
4. お気に入り一覧

## 8. スコープ外

- 外部レシピAPIとの連携
- 栄養価計算
- 買い物リスト連携(将来的な拡張候補としては残してよい)
